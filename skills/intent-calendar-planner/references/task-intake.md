# Task Intake

Use these questions only when the answer is not already clear from the conversation, the vault, or the calendar.

## Minimum Questions Before Scheduling A Task

1. What is the concrete goal of this block?
2. Why does it matter today or now?
3. What counts as done for this block?
4. How long should the block be?
5. What are the first meaningful implementation steps?
6. Which project or area does it support?
7. Are there any constraints, dependencies, locations, or energy requirements?

## High-Signal Follow-Up Phrasing

- `What exactly do you want finished by the end of this block?`
- `Why does this deserve calendar time today?`
- `If this block goes well, what is materially different afterward?`
- `How long do you want to protect for this?`
- `What are the first 2-5 steps you would actually follow once you start?`
- `Which project does this belong to?`
- `Is there any timing, location, or energy constraint I should respect?`

## When Not To Ask

Do not ask a question just because the template has a slot for it. If the project file or the user's message already answers it with high confidence, use that information and move on.

## When To Push Back

Push back briefly when:

- the task is too vague to schedule responsibly
- the user is trying to schedule something that should first be clarified
- the duration is clearly unrealistic
- the task appears disconnected from meaningful priorities without a good reason
